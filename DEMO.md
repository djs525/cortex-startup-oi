# Cortex — Demo Script

---

## Before you start

- [ ] All 5 databases built and populated with sample data
- [ ] 2+ deals with `Last Contact` older than 7 days
- [ ] 1+ OKR with Status = `At Risk`
- [ ] 1+ investor in `Due Diligence` with `Last contact` older than 7 days
- [ ] Notion MCP connected, Cortex system prompt loaded
- [ ] Company Pulse dashboard open in Notion

---

## Demo 1 — `run Cortex`

> *"Monday morning. Full company picture. Two words."*

```
run Cortex
```

Claude reads all 5 databases, scores health Green/Yellow/Red, creates a fully written weekly review in Notion — all 8 sections, dated, Generated by Claude checked.

**Switch to Notion.** Show the new page in Weekly Review DB.

> *"I didn't write a word of that. Cortex told me the two things I need to fix today."*

---

## Demo 2 — `stale deals`

> *"Any deals gone quiet? One command."*

```
stale deals
```

Claude scans the pipeline, computes exact days silent per deal, flags anything 7+ days, and writes a personalised follow-up email per deal — referencing specific context from the Notes field.

**Show the alert page in Notion.** Point to the drafted email.

> *"$24k deal, 12 days silent. Cortex wrote the follow-up. I just hit send."*

---

## Demo 3 — `activate Engineering`

> *"Six months later. First engineers hired. One command."*

```
activate Engineering
```

Cortex explains the module, provides the full database spec, walks through setup, and confirms what it will now monitor.

> *"Same system. Same commands. Smarter because the team got bigger."*

---

## Demo 4 — `health check`

> *"Just set up Cortex for the first time. Is everything connected?"*

```
health check
```

Cortex reads each active database and reports ✅ Accessible / ⚠️ Empty / ❌ Inaccessible — with exact fixes for anything broken.
