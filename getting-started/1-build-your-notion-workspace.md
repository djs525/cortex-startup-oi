# Step 1 — Build your Notion workspace

**Time:** ~30–40 minutes · No technical experience required

---

## Setup

1. Create a dedicated Notion workspace — don't use your personal one
2. Name it `[Your Company] — Cortex`
3. Create one parent page called `Cortex` with these child pages:

```
📁 Cortex
├── 🏠 Company Pulse
├── 🎯 OKR Tracker
├── 🛒 Sales Pipeline
├── 💼 Investor CRM
├── 🧑 Hiring Pipeline
└── 📋 Weekly Review
```

---

## Build the 5 databases

Open each page → type `/database` → choose **Full page database**.

### OKR Tracker DB

| Property | Type | Notes |
|---|---|---|
| Objective | Title | The goal name |
| Level | Select | Company · Team · Individual |
| Status | Select | On Track (green) · At Risk (yellow) · Off Track (red) · Done (blue) |
| Quarter | Select | Q1 · Q2 · Q3 · Q4 |
| Owner | Person | Accountable person |
| Progress % | Number | Ring format, 0–100 |
| Notes | Text | Context and blockers |
| Linked Deals | Relation | → Sales Pipeline DB (add after building Sales) |
| Linked Roles | Relation | → Hiring Pipeline DB (add after building Hiring) |

### Sales Pipeline DB

| Property | Type | Notes |
|---|---|---|
| Deal Name | Title | Company or person |
| Stage | Select | Lead · Qualified · Proposal · Negotiation · Closed won · Closed lost |
| Deal Value | Number | Estimated $ value |
| Close date | Date | Expected close |
| Last Contact | Date | ⭐ Critical — Cortex scans this for stale deals |
| Contact Name | Text | Primary contact |
| Owner | Person | Deal owner |
| Loss reason | Select | Price · Missing Feature · Competitor · Timing · No response |
| Linked OKR | Relation | → OKR Tracker DB |
| Notes | Text | Context, objections, next steps |

Add a **Board view grouped by Stage** after building.

### Investor CRM DB

| Property | Type | Notes |
|---|---|---|
| Investor / Firm | Title | Name of investor or fund |
| Stage | Select | Backlog · Intro requested · Meeting scheduled · Due diligence · Term sheet · Passed · Closed |
| Tier | Select | Lead Investor · Follow-on · Angel · Strategic |
| Last contact | Date | ⭐ Cortex flags overdue follow-ups |
| Check size | Number | Target amount |
| Warm intro via | Text | Who introduced you |
| Notes | Text | Meeting notes, asks, next steps |

### Hiring Pipeline DB

| Property | Type | Notes |
|---|---|---|
| Role | Title | Job title |
| Stage | Select | Open · Sourcing · Screening · Interviews · Offer · Hired · Closed |
| Team | Select | Sales · Engineering · Product · Marketing · People · Finance · Ops |
| Priority | Select | Critical · High · Normal |
| Open date | Date | ⭐ Cortex flags roles open 30+ days |
| Linked OKR | Relation | → OKR Tracker DB |
| Candidate count | Number | Pipeline size |

### Weekly Review DB

| Property | Type | Notes |
|---|---|---|
| Week of | Title | e.g. "Week of Apr 7, 2026" |
| Status | Select | Draft · Published |
| Overall health | Select | Green · Yellow · Red |
| Pipeline value | Number | Total active deal value |
| OKRs on Track | Number | Count of On Track OKRs |
| Generated by Claude | Checkbox | Was this AI-generated? |

Create a **default template** with these 8 headings:
```
## Wins this week
## Blockers
## OKR pulse
## Sales snapshot
## Hiring update
## Investor update
## Decisions made
## Next week priorities
```

---

## Wire the relations

**Critical — without this, Cortex can't connect the dots.**

- Sales Pipeline → `Linked OKR` → OKR Tracker DB
- Hiring Pipeline → `Linked OKR` → OKR Tracker DB
- OKR Tracker → add Rollup: **Sum of Deal Value** from Linked Deals

> When you add a relation, Notion creates a reverse relation automatically. Rename it something sensible.

---

## Add sample data

Minimum 3–5 rows per database. For the demo to work:
- 2 deals with `Last Contact` older than 7 days
- 1 OKR with Status = `At Risk`
- 1 investor in `Due Diligence` with `Last contact` older than 7 days

---

## Build the Company Pulse dashboard

Open Company Pulse → use `/linked view` to embed a view from each database. This is Cortex's master view and the first thing judges see.
