## Core

**`run Cortex`** — also: `weekly review`, `Monday briefing`
Reads all active databases. Scores health:
- 🟢 Green — 3+ OKRs On Track, no deals stale 10+ days, no Lead Investor overdue
- 🟡 Yellow — 1–2 OKRs At Risk OR deal silent 7–14 days OR investor overdue
- 🔴 Red — OKR Off Track OR Negotiation deal silent 14+ days OR Lead Investor silent 14+ days

Creates `Week of [date]` in Weekly Review DB — Draft, Generated by Claude ✓, all 8 sections.

**`full briefing`** — runs all 5 workflows. Use before board meetings or quarterly reviews.

**`company status`** — 4–6 sentence spoken summary. No Notion write. Use mid-week.

---

## Pipeline

**`stale deals`** — also: `check the pipeline`
Flags 🔴 Critical (10+ days at Negotiation/Proposal) and 🟡 Watch (7+ days anywhere). Creates alert page with personalised follow-up emails using the deal's Notes field context.

**`pipeline health`** — pipeline value, stage distribution, weighted forecast. Chat only.

**`loss analysis`** — also: `why are we losing deals`
Patterns in Closed lost deals — product implications, positioning implications, re-engage candidates. Any Missing Feature loss is flagged as a re-engage candidate when that feature ships.

---

## Investor

**`investor check`** — also: `fundraising update`
Flags 🔴 Urgent (Lead Investor in Due Diligence/Term Sheet, silent 7+ days) and 🟡 Follow up (any active investor silent 10+ days). Creates alert with what they asked for, whether it was delivered, and a drafted follow-up.

**`draft investor update`** — reads OKRs + pipeline + hiring. Full monthly investor update — subject, summary, metrics, priorities, ask. Created as Draft in Weekly Review DB.

---

## Team

**`hiring check`** — also: `team gaps`
Flags 🔴 Stalling (Critical role open 30+ days) and 🟡 Slow (High priority open 21+ days OR <3 candidates). Returns bottleneck diagnosis and next action per role.

**`OKR check`** — all OKRs with status, progress %, owner, root cause from Notes. Chat only.

---

## Modules

**`activate [MODULE]`** — MODULE = `Engineering` `Product` `People` `Finance` `Marketing`
Walks through: build database → wire `Linked OKR` → copy data source ID → update system prompt. Confirms activation and lists new commands unlocked.

**`deactivate [MODULE]`** — pauses the module. Data preserved. Update system prompt: `ACTIVE` → `INACTIVE`.

**`module status`** — all 9 modules with state and what each inactive one unlocks.

---

## Setup

**`setup Cortex`** — first-time onboarding. Checks MCP connection, collects all database IDs, runs `health check`, then `run Cortex` to confirm everything works.

**`health check`**
- ✅ Accessible — readable with rows
- ⚠️ Empty — accessible but no data yet
- ❌ Inaccessible — wrong ID, page not shared, or placeholder not replaced

For each ❌, Cortex provides the exact fix.
