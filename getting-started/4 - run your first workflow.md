# Step 4 — Run your first workflow

**Time:** 2 minutes · This is where it all comes together.

---

## Option A — Automated setup (recommended)

If you want Claude to build the workspace for you, skip Steps 1–3 entirely and just type:

```
setup Cortex
```

Claude will:
1. Create all 6 parent pages with icons
2. Create all 5 databases inside them
3. Populate each database with realistic sample data — including stale deals and at-risk OKRs ready for the demo
4. Create the Weekly Review template with all 8 sections
5. Build the Company Pulse dashboard structure
6. Generate a personalised setup checklist with your exact database IDs already filled in

**You'll still need to do 3 manual steps in Notion (~5 minutes):**
- Add the property schemas to each database (select options, dates, etc.)
- Wire the relations: Sales → OKR, Hiring → OKR
- Share each page with the Cortex MCP integration

Claude will give you the exact instructions for each step with screenshots references.

Then type `run Cortex` to generate your first weekly review.

---

## Option B — Manual setup then run

If you followed Steps 1–3 manually, just type:

```
run Cortex
```

Claude reads all 5 databases, scores health 🟢🟡🔴, and creates a fully written weekly review in Notion — all 8 sections, dated, Generated by Claude checked.

Switch to Notion. Open Weekly Review. The page is there.

---

## Troubleshoot

**"Object not found"** — database not shared with the integration. Open it in Notion → `...` → Connect to → select your integration.

**Claude responds conversationally** — system prompt not loaded. Check Custom Instructions or Project Instructions, open a fresh session.

**Databases return empty** — run `health check` to see exactly what's accessible.
